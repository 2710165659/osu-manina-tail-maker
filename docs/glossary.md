# 术语对齐与领域定义 (Glossary)

> 本文档对齐项目中涉及的全部术语，明确每个概念在 osu! mania 皮肤系统与本品内的准确定义。

---

## 1. osu!mania 皮肤基础概念

| 术语 | 英文 | 对应皮肤文件 | 本品是否涉及 |
|------|------|-------------|-------------|
| 落下音符头部 | Note Head | `mania-note1.png` ~ `mania-noteN.png` | **不涉及** |
| 长按身体 / 投皮 | Hold Body | `mania-note1H.png` ~ `mania-noteNH.png` | **唯一生成目标** |
| 长按尾端 | Note Tail / Hold Tail | `mania-note1L.png` ~ `mania-noteNL.png` | **不涉及** |

> **核心结论**：本品是一个 **纯粹的 `mania-noteH.png` 生成器**。不处理 Note Head、Note Tail。

---

## 2. 投皮 (Hold Body / Tóu Pí)

**定义**：osu!mania 中长按音符（Hold Note / LN）按下后持续显示的纵向竖条图像。

**核心原理**：游戏拿到 `mania-noteH.png` 后，从 Note Head 位置开始向下绘制。如果图片本身很长（如 32800px），游戏全程只显示这一张图的连续区域，**不会出现平铺/重复**，视觉上是平滑的。当长按结束时，图片不再显示，玩家看到的是尾端自然消失，由投皮顶端来承担结束瞬间的视觉表现。

**本品唯一输出**：每个 `mania-noteNH.png` 文件。

---

## 3. 投皮图片结构

下图是整张 `mania-noteH.png` 的二维剖面。水平方向两侧为留白区，中间为内容区；垂直方向从上到下分为投的长度、顶端、身体三段。

```
          ←────────── 图片宽度 (Image Width)，默认 40px ──────────→

  ┌──────┬──────────────────────────────┬──────┐
  │ 透明  │          透明                 │ 透明  │  ──┐
  │ 透明  │          透明                 │ 透明  │    ├── 投的长度 (Throw Length)
  │ 透明  │          透明                 │ 透明  │  ──┤   顶部纯透明，单位 px
  ├──────┼──────────────────────────────┼──────┤  ──┤   控制身体可见的延迟
  │ 透明  │      顶端 / Cap               │ 透明  │    │
  │ 透明  │   球皮/菱形/矩形/渐变          │ 透明  │    ├── 顶端高度 (Cap Height)
  ├──────┼──────────────────────────────┼──────┤  ──┤   长按结束的视觉关键
  │ 透明  │                              │ 透明  │    │
  │ 透明  │      身体 / Body              │ 透明  │    │
  │ 透明  │    · 填充色 (Fill)            │ 透明  │    ├── 身体 (Body)
  │ 透明  │    · 边框 (Border)            │ 透明  │    │   一直延伸到图片底部
  │ 透明  │                              │ 透明  │    │
  │ 透明  │           ...                 │ 透明  │    │
  │ 透明  │           ...                 │ 透明  │    │
  │ 透明  │           ...                 │ 透明  │  ──┘
  └──────┴──────────────────────────────┴──────┘
  ↑      ↑                             ↑      ↑
 留白    │                            留白     │
(Margin) 内容区 (Content)             (Margin)  底部不留白
(左右对称)                                        (左右对称)
```

> **关键规则**：
> - 留白 = 图片左右两侧对称的透明边距，**贯穿全图**（从顶部到底部每一行都生效），左右值始终相等
> - 投的长度 = 图片**顶部**的透明行数（含留白列）
> - **底部不留白**：身体内容一直延伸到图片最后一行

### 3.1 图片宽度 (Image Width)

整张图片的水平宽度，默认 40px，用户可自定义。

### 3.2 留白 (Side Margin)

图片**左右两侧**的透明边距，**左右始终对称**（同一数值同时控制两侧）。留白让实际内容区域变窄，视觉上投皮更纤细。

- `留白 (Margin)`：左右两侧各自的透明像素数。内容区宽度 = 图片宽度 − 留白 × 2
- **底部不留白**：身体内容一直延伸到图片最底部
- **顶部也不留白**：顶部透明度由"投的长度"控制，见下文

### 3.3 投的长度 (Throw Length)

图片**最顶部**的纯透明区域，单位 px。这段透明像素在游戏中表现为从 Note Head 按下到身体可见之间的视觉间隔。

- 数值越大 → 按下后等越久才看到身体
- 数值越小 → 身体几乎立刻出现
- 设为 0 → 身体紧贴 Note Head

### 3.4 顶端 / 投皮头 (Cap)

身体可见部分的**最上方区域**，是投皮的视觉核心。长按音符结束时，玩家最后看到的就是顶端形状消失，因此顶端决定了"结束感"。

支持的顶端形状（预设 + 自定义）：

| 预设名 | 形状 | 说明 |
|--------|------|------|
| **球皮 (Ball)** | 半圆 | 顶端呈半圆形，圆弧朝上，模拟球状投皮。社区最常见的样式。 |
| **菱形 (Diamond)** | 半菱形 | 顶端呈半个菱形（等腰三角形的上半部），尖锐。 |
| **矩形 (Rectangle)** | 矩形拉伸 | 普通的矩形，顶端水平截断，无特殊形状。 |
| **渐变 (Gradient)** | 矩形+透明度渐变 | 矩形顶端区，从上到下透明度递减（从不透明到身体色），形成渐入效果。 |

### 3.5 身体 (Body)

顶端以下、图片底部以上的全部区域。

- **填充 (Fill)**：身体的底色，可设颜色与透明度
- **边框 (Border)**：身体外轮廓的描边，可设颜色与透明度。边框独立于填充，可以单独开关调整

## 4. 属性维度总览

| 属性 | 作用范围 | 可配项 |
|------|---------|--------|
| 图片宽度 | 整张图 | 像素值（默认 40） |
| 图片高度 | 整张图 | 像素值（默认 32800） |
| 图片名称 | 导出文件 | 字符串（不含扩展名） |
| 留白 | 水平两侧（对称） | px |
| 投的长度 | 顶部透明区 | px |
| 顶端形状 | Cap 区域 | 球皮 / 菱形 / 矩形 / 渐变 |
| 顶端尺寸 | Cap 区域 | 高度 px |
| 身体填充色 | Body 填充 | RGBA（含透明度） |
| 身体边框色 | Body 边框 | RGBA（含透明度） |
| 边框粗细 | Body 边框 | px |
| 整体透明度 | 整张图非透明区 | 0–100% |

---

## 5. 预设方案 (Presets)

预设是一组上述属性的快照。用户可以选择一个预设快速生成，也可以基于预设微调。

预设涵盖：
1. 球皮 — 半圆顶端 + 细边框 + 半透明身体
2. 菱形皮 — 半菱形顶端 + 无边框
3. 渐变皮 — 矩形顶端渐变 + 无边框
4. 纯色皮 — 无特殊顶端 + 纯色填充
5. （更多预设可在开发中扩展）
