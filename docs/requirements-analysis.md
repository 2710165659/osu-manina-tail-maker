# 详细需求分析 (Requirements Analysis)

> 版本：v1.0 | 日期：2026-06-01

---

## 1. 项目定位

**osu-manina-tail-maker** 是一个 **Windows 桌面工具**，专注生成 osu!mania 模式的 `mania-noteH.png`（长按身体/投皮）图片。用户通过可视化界面调控各项参数，实时预览效果，一键导出成品。

### 1.1 核心价值

- **告别手工**：不需要在 PS 里手动拉 32800px 高的图层
- **实时预览**：改参数即时看效果，不靠脑补
- **参数化调优**：每个视觉属性独立可调，组合灵活
- **预设 + 自定义**：快速套用预设，或从零搭建

### 1.2 不做什么

- 不生成 Note Head（`mania-note1.png`）
- 不生成 Note Tail（`mania-note1L.png`）
- 不做整套皮肤的打包/管理
- 不处理其他 osu! 模式（standard / catch / taiko）

## 2. 功能需求

### 2.1 图片参数配置

#### 2.1.1 图片尺寸

| 参数 | 类型 | 默认值 | 范围 | 说明 |
|------|------|--------|------|------|
| 图片宽度 | int | 40 px | 1–4096 | 整图水平像素数 |
| 图片高度 | int | 32800 px | 1–65535 | 整图垂直像素数 |

> 默认 40×32800 是 mania 4K 投皮的社区事实标准。用户可为其他键数（5K/6K/7K 等）调整宽度。

#### 2.1.2 留白

| 参数 | 类型 | 默认值 | 范围 | 说明 |
|------|------|--------|------|------|
| 留白 | int | 0 px | 0 – floor((图片宽度-1)/2) | 左右两侧对称的透明边距 |

- 约束：`留白 × 2 < 图片宽度`（至少保留 1px 内容区）
- 留白区域始终**全透明**
- 左右始终对称，不可单独设

#### 2.1.3 内容区域

内容区宽度 = `图片宽度 - 留白 × 2`。所有可见元素（顶端 + 身体）绘制在内容区内。

#### 2.1.4 投的长度

| 参数 | 类型 | 默认值 | 范围 | 说明 |
|------|------|--------|------|------|
| 投的长度 | int | 0 px | 0 – (图片高度-1) | 图片顶部纯透明像素数 |

- 这段区域是**全透明**的，不绘制任何内容
- 位于图片最顶部，Cap 之前

### 2.2 顶端 (Cap) 配置

顶端是投皮最上方可见部分，决定了长按结束时的视觉呈现。

#### 2.2.1 顶端形状

| 形状 | ID | 描述 |
|------|-----|------|
| 矩形 | `rect` | 普通矩形，无特殊处理。水平齐平开始。 |
| 球皮（半圆） | `ball` | 顶端呈上半圆弧，圆弧最高点对齐 Cap 起始线，两侧降到身体两侧。 |
| 菱形（半菱形） | `diamond` | 顶端呈等腰三角形的上半部，顶点对齐 Cap 起始线，两侧斜线延伸至身体两侧。 |
| 渐变 | `gradient` | 矩形顶端，但从上到下透明度渐变：顶端起始线处全透明 → Cap 结束处为身体填充色（带身体透明度）。 |

#### 2.2.2 顶端尺寸

| 参数 | 类型 | 默认值 | 范围 | 说明 |
|------|------|--------|------|------|
| 顶端高度 | int | 30 px | 0 – min(500, 图片高度-投的长度) | Cap 区域的垂直像素数 |

- 顶端高度 > 0 时，该区域绘制选定形状
- 顶端高度 = 0 时，跳过 Cap，身体直接从"投的长度"之后开始

#### 2.2.3 顶端颜色与透明度

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 顶端颜色 | color | 与身体填充色一致 | 顶端区域的填充色（RGBA），默认跟随身体 |
| 顶端独立透明度 | bool | false | 开启后可独立设置顶端的透明度 |
| 顶端透明度 | uint8 | 255 | 仅独立模式时生效，0–255 |

### 2.3 身体 (Body) 配置

身体是从 Cap 结束线到图片底部之间的全部区域。

#### 2.3.1 填充

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 身体填充色 | color | `#FFFFFF` | 填充颜色（RGB） |
| 身体透明度 | uint8 | 255 | 0（全透明）– 255（不透明） |

- 身体覆盖内容区的全部宽度
- 身体从 Cap 结束线一直延伸到图片底部

#### 2.3.2 边框

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 显示边框 | bool | true | 是否绘制身体边框 |
| 边框颜色 | color | `#000000` | 边框颜色（RGB） |
| 边框透明度 | uint8 | 255 | 0（全透明）– 255（不透明） |
| 边框粗细 | int | 1 px | ≥ 1，边框向内绘制 |

- 边框沿身体内容区外围绘制（上下左右四边）
- 边框在填充之上绘制

### 2.4 整体透明度

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| 整体透明度 | uint8 | 255 | 作用于所有非透明区域的乘算透明度 |

> 整体透明度与各区域的独立透明度是乘法关系。最终 alpha = 区域透明度 × 整体透明度 / 255。

### 2.5 预览功能

#### 2.5.1 实时预览

- 用户修改任何参数，预览区即时更新（debounce ≤ 200ms）
- 预览采用**纵向滚动**，可上下滚动查看整张图
- 预览图上叠加标注线，标示"投的长度"结束线、Cap 结束线

#### 2.5.2 缩放

| 操作 | 说明 |
|------|------|
| 适应宽度 | 预览宽度缩放到预览区宽度 |
| 适应高度 | 预览高度缩放到预览区高度（图片会被极度压缩） |
| 1:1 像素 | 实际像素大小查看，可拖拽平移 |
| 滚轮缩放 | 用户自由缩放 |

- 默认：**适应宽度**，方便查看全长

### 2.6 图片导出

| 参数 | 说明 |
|------|------|
| 图片名称 | 用户输入，不含扩展名。如 `mania-note1H` |
| 输出格式 | PNG（必须支持透明通道 RGBA） |
| 输出路径 | 用户选择目录 |

- 导出文件命名：`{图片名称}.png`
- 重复导出时提示覆盖确认

### 2.7 预设管理

#### 2.7.1 内置预设

| 预设名 | 顶端 | 边框 | 描述 |
|--------|------|------|------|
| 球皮-标准 | ball, 30px | 有, 1px | 半圆头 + 细边框 |
| 菱形-无边框 | diamond, 40px | 无 | 尖锐菱形头 + 纯填充 |
| 渐变-无边框 | gradient, 60px | 无 | 矩形渐变头，从上淡入 |
| 纯色-无头 | rect, 0px | 无 | 无顶端，直接纯色身体 |
| 纯色-带边框 | rect, 0px | 有, 2px | 无顶端，纯色身体 + 粗边框 |

#### 2.7.2 预设操作

- **加载预设**：选择一个预设，所有参数替换为预设值，预览更新
- **保存为预设**：当前参数保存为新预设（存本地）
- **删除预设**：删除用户自建预设（内置预设不可删）
- **重置**：恢复到默认参数

### 2.8 撤销/重做

- 参数修改支持撤销（Undo）和重做（Redo）
- 撤销栈深度默认 50 步

## 3. 非功能需求

### 3.1 性能

| 指标 | 要求 |
|------|------|
| 参数修改 → 预览更新延迟 | ≤ 200ms（含 debounce） |
| 首次生成 40×32800 图片耗时 | ≤ 2s |
| 导出保存耗时 | ≤ 3s |
| 内存占用（运行时） | ≤ 500MB |

### 3.2 图片质量

- 输出 PNG 必须为 RGBA 8-bit 色深
- 抗锯齿：顶端形状边缘必须抗锯齿，不允许锯齿状
- 透明通道必须保留，不得被压平

### 3.3 兼容性

| 目标 | 版本 |
|------|------|
| Windows | Windows 10 及以上（x64） |
| osu! 版本 | stable / lazer 均兼容（PNG 格式通用） |

### 3.4 可用性

- 界面语言：中文（简体）
- 所有输入框支持手动输入 + 方向键微调
- 关键参数提供 tooltip 说明
- 输入非法值时有校验提示

## 4. 用户故事

### US-01：快速生成一张球皮

> 作为一名 mania 玩家，我想选择一个球皮预设，调整投的长度，然后直接导出，这样我就能在 30 秒内完成一张投皮。

1. 打开程序
2. 点击预设「球皮-标准」
3. 预览区即时显示效果
4. 调一下投的长度（比如从 0 改成 10）
5. 输入文件名，点导出
6. 把生成的 PNG 放进 skin 文件夹

### US-02：精细调参

> 作为一个对投皮有明确想法的制皮者，我想逐个参数调节，从零搭建一张投皮。

1. 设图片宽度 50（给 5K 用）
2. 设留白 3
3. 设投的长度 8
4. 选顶端形状「菱形」，高度 50
5. 身体填充色选 `#FF6B9D`，透明度 200
6. 边框打开，颜色 `#FFFFFF`，透明度 180，粗细 2
7. 实时预览随每步变化
8. 满意后导出

### US-03：保存并复用自定义预设

> 作为制皮者，我调好了一套参数后想保存下来，以后反复使用。

1. 调好参数
2. 点击「保存为预设」，命名为「我的粉色菱形」
3. 下次打开程序，预设列表里出现该项
4. 点击即加载

### US-04：预览检查细节

> 作为制皮者，我想放大预览图检查顶端抗锯齿和边框细节。

1. 滚轮放大预览至 4x
2. 拖拽平移查看顶端区域
3. 确认边缘平滑无锯齿
4. 满意后导出

## 6. 数据流

```
参数配置 ──→ 渲染引擎 (Rust) ──→ 像素缓冲区 ──→ PNG 编码 ──→ 文件输出
                │
                └──→ 缩略图 ──→ 前端预览 (Vue/Canvas)
```

1. 用户在 Vue 界面调整参数
2. 参数通过 Tauri 命令传递给 Rust 后端
3. Rust 后端执行像素级渲染，生成 RGBA 像素缓冲区
4. 同时生成预览用缩略图回传前端
5. 导出时，Rust 将完整缓冲区编码为 PNG 写入磁盘
